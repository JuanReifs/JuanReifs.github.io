Memoria del Proyecto

A continuación se presenta la memoria completa del proyecto Portfolio Profesional, donde se detallan todos los aspectos del desarrollo, implementación y resultados obtenidos durante el primer trimestre del ciclo de Desarrollo de Aplicaciones Multiplataforma.

1. Resumen Ejecutivo

Título del Proyecto: Portfolio Profesional - Presentación Digital Interactiva

Autor: Juan Reifs García

Centro Educativo: CES Juan Pablo II

Asignatura: Lenguajes de Marcas y Sistemas de Gestión de Información

Fecha: Febrero 2026

Este proyecto consiste en el desarrollo de un sitio web portfolio profesional utilizando HTML5, CSS3 y JavaScript. El objetivo principal es crear una plataforma digital moderna que sirva como carta de presentación profesional, integrando CV digital interactivo, carta de presentación personalizada, portfolio de prácticas realizadas durante el curso, y documentación técnica completa del proyecto.

El portfolio no solo cumple con los requisitos académicos establecidos, sino que también representa una herramienta práctica para mi futura búsqueda de prácticas profesionales, demostrando competencias técnicas y capacidad de desarrollo frontend.

2. Objetivos del Proyecto

  • Objetivo Principal: Crear un portfolio web profesional y completamente funcional que demuestre las competencias adquiridas en el primer trimestre del ciclo DAM
  • Demostrar dominio técnico: Aplicar conocimientos de HTML5, CSS3 y JavaScript para crear una experiencia de usuario moderna y fluida
  • Presentación profesional: Desarrollar una plataforma efectiva para presentar información académica, técnica y profesional de forma atractiva
  • Diseño responsive: Implementar un diseño completamente adaptable que funcione correctamente en dispositivos móviles, tablets y ordenadores
  • UI/UX moderno: Aplicar principios de diseño de interfaz y experiencia de usuario, incluyendo animaciones, transiciones y elementos interactivos
  • Optimización y rendimiento: Garantizar tiempos de carga rápidos y navegación fluida mediante código optimizado
  • Documentación exhaustiva: Registrar todo el proceso de desarrollo, decisiones técnicas y aprendizajes obtenidos

3. Metodología de Trabajo

El desarrollo del proyecto se ha estructurado siguiendo una metodología iterativa en cuatro fases principales:

Fase 1

Planificación y Diseño

Análisis de requisitos del proyecto y definición de estructura del sitio. Creación de wireframes y mockups iniciales. Selección de paleta de colores (#2563eb azul primario, #f59e0b acento dorado) y tipografías (Playfair Display para títulos, Source Sans 3 para cuerpo). Planificación de la arquitectura de información y flujo de navegación.

Fase 2

Desarrollo de Estructura HTML

Creación de la estructura semántica del sitio utilizando HTML5. Implementación de todas las secciones requeridas: Hero, Carta de Presentación, CV Digital, Portfolio de Prácticas, Temario y Memoria. Uso apropiado de etiquetas semánticas (nav, section, article, aside, footer) para mejorar accesibilidad y SEO.

Fase 3

Estilado CSS y Animaciones

Implementación de sistema de variables CSS para mantenibilidad. Aplicación de estilos visuales, diseño responsive mediante Flexbox y CSS Grid. Creación de animaciones CSS (flotar, fade-in, hover effects). Optimización para diferentes tamaños de pantalla mediante media queries. Desarrollo de elementos decorativos animados (círculos flotantes, partículas, formas geométricas).

Fase 4

JavaScript y Testing Final

Implementación de funcionalidad JavaScript para navegación dinámica y detección de sección activa. Testing exhaustivo en múltiples navegadores (Chrome, Firefox, Safari, Edge) y dispositivos (móvil, tablet, desktop). Corrección de bugs encontrados, optimización de rendimiento y ajustes finales de diseño. Validación de código HTML y CSS.

4. Tecnologías Utilizadas

HTML5

Lenguaje de marcado estructural. Uso extensivo de etiquetas semánticas (header, nav, section, article, aside, footer) para mejorar accesibilidad y SEO. Implementación de atributos meta para responsive design.

CSS3

Hojas de estilo en cascada para diseño visual completo. Variables CSS (custom properties) para gestión de colores y espaciados. Flexbox y CSS Grid para layouts complejos y responsivos. Animaciones y transiciones CSS. Pseudo-elementos para decoraciones.

JavaScript (Vanilla)

Lenguaje de programación para interactividad. Detección de scroll para navegación activa. Manipulación del DOM. Event listeners para interacciones de usuario. Código limpio sin dependencias de librerías externas.

Google Fonts

Servicio de tipografías web profesionales. Playfair Display (serif) para títulos y encabezados. Source Sans 3 (sans-serif) para texto de cuerpo. Optimización de carga con preconnect.

Visual Studio Code

Editor de código principal utilizado para el desarrollo. Extensiones utilizadas: Live Server, Prettier, HTML CSS Support, Auto Rename Tag.

Git & GitHub

Sistema de control de versiones para seguimiento de cambios. Repositorio en GitHub para almacenamiento y respaldo del código. Commits descriptivos documentando el progreso del desarrollo.

5. Estructura del Proyecto

El sitio web se compone de las siguientes secciones principales, cada una diseñada con un propósito específico:

  • Navegación Principal: Barra de navegación fija con efecto glassmorphism, indicador de sección activa mediante JavaScript, y animaciones hover en los enlaces. Permite acceso rápido a todas las secciones del portfolio.
  • Hero Section (Inicio): Sección de presentación impactante con foto de perfil animada, título con gradiente animado, descripción breve, enlaces a redes sociales (LinkedIn, GitHub, Instagram), y llamadas a la acción (CTAs) para ver CV y proyectos.
  • Carta de Presentación: Documento formal personalizado detallando mi visión como desarrollador, competencias técnicas (Java, JavaScript, SQL, HTML, CSS, XML, C#, C++, Shell), enfoque en IA y automatización, habilidades interpersonales certificadas, y objetivos profesionales.
  • CV Digital Interactivo: Currículum completo con diseño en dos columnas. Sidebar con foto, datos de contacto, habilidades técnicas con barras de progreso animadas, e idiomas. Sección principal con perfil profesional, formación académica en timeline, experiencia con proyectos personales, y certificaciones.
  • Portfolio de Prácticas: Grid responsive mostrando 6 proyectos realizados: Sistema de Gestión de Archivos (Python), Calculadora Web (HTML/CSS/JS), Página Web Navideña (proyecto grupal), Algoritmos de Ordenación (Java), Portfolio Personal (este proyecto), y Aplicación To-Do List. Cada tarjeta incluye descripción, tecnologías usadas, y enlaces a GitHub.
  • Temario del Curso: Documentación de contenidos estudiados organizados por asignatura: Sistemas Informáticos, Programación (POO con Java), Lenguajes de Marcas (HTML/CSS/JS), ECEM (empleabilidad), SEO, e IPE (derechos laborales).
  • Memoria del Proyecto: Esta sección actual, documentando completamente el proceso de desarrollo, tecnologías utilizadas, metodología aplicada, y conclusiones obtenidas.

6. Características Técnicas Destacadas

Sistema de Variables CSS: Implementación completa de custom properties para gestión centralizada de colores, tipografías, espaciados, transiciones y sombras. Facilita el mantenimiento y futuras modificaciones del diseño.

Animaciones Avanzadas: Círculos decorativos flotantes con movimiento orgánico mediante keyframes complejos. Animaciones de entrada (fade-in, slide-in) para elementos al cargar la página. Efectos hover suaves en todos los elementos interactivos. Partículas animadas en el fondo usando pseudo-elementos.

Optimización de Rendimiento: CSS optimizado con selectores eficientes. Uso de transform para animaciones (mejor rendimiento que left/top). Preconnect para Google Fonts reduciendo tiempo de carga. Código JavaScript mínimo y eficiente sin librerías pesadas.

Accesibilidad y Semántica: Estructura HTML semántica correcta para lectores de pantalla. Contraste de colores adecuado (WCAG AA). Tamaños de texto legibles y escalables. Navegación mediante teclado funcional.

7. Proceso de Desarrollo y Desafíos

Durante el desarrollo del proyecto, se presentaron diversos desafíos técnicos que contribuyeron significativamente al aprendizaje:

  • Animaciones de Fondo: Crear círculos flotantes con movimiento natural fue un reto. Se resolvió utilizando múltiples keyframes con transformaciones combinadas (translate, scale, rotate) y timings diferentes para cada círculo, logrando un efecto orgánico y no repetitivo.
  • Navegación Dinámica: Implementar la detección de sección activa requirió calcular correctamente el offset de cada sección y detectar el scroll. Se utilizó JavaScript vanilla con event listeners optimizados para evitar problemas de rendimiento.
  • Responsive Design Consistente: Asegurar que el diseño se viera bien en todos los tamaños de pantalla requirió múltiples iteraciones. Se utilizó un enfoque mobile-first y testing en dispositivos reales para validar el diseño.
  • Performance de Animaciones: Las animaciones CSS inicialmente causaban lag en dispositivos menos potentes. Se optimizó usando will-change y transform en lugar de propiedades que fuerzan reflow/repaint.
  • Organización del Código CSS: Con más de 1000 líneas de CSS, mantener el código organizado fue crucial. Se implementó un sistema de secciones comentadas y se siguió la metodología BEM parcialmente para nomenclatura de clases.

8. Aprendizajes Clave

Este proyecto ha sido fundamental para consolidar y expandir mis conocimientos en desarrollo web frontend:

Variables CSS (Custom Properties): Comprensión profunda de cómo las variables CSS facilitan el mantenimiento y permiten cambios globales de diseño de forma eficiente. Aprendí a estructurar un sistema de diseño coherente.

Animaciones CSS: Capacidad para crear animaciones complejas usando keyframes, transitions y transforms. Entendimiento de conceptos como timing-functions, animation-delay y animation-fill-mode.

JavaScript DOM: Manipulación efectiva del DOM para crear interactividad. Uso de querySelector, addEventListener, classList y cálculos de posición (offsetTop, scrollY) para la navegación dinámica.

Responsive Design: Estrategias efectivas para diseño adaptable: mobile-first approach, breakpoints estratégicos, unidades relativas (rem, em, %), y testing en dispositivos reales.

Semántica HTML: Importancia de usar las etiquetas correctas (nav, section, article, aside) no solo para estructura sino para accesibilidad, SEO y mantenibilidad del código.

9. Validación y Testing

Se realizaron pruebas exhaustivas para garantizar la calidad del proyecto:

  • Validación W3C: Código HTML validado sin errores en el validador oficial del W3C. CSS validado cumpliendo estándares CSS3.
  • Testing Cross-Browser: Pruebas en Chrome, Firefox, Safari y Edge. Verificación de compatibilidad de animaciones CSS y JavaScript en todos los navegadores.
  • Responsive Testing: Pruebas en dispositivos reales: iPhone 12, Samsung Galaxy S21, iPad Pro. Uso de herramientas de desarrollo Chrome para simular diferentes resoluciones.
  • Performance Testing: Lighthouse audit con puntuaciones: Performance 95+, Accessibility 90+, Best Practices 100, SEO 90+.
  • Usability Testing: Pruebas con usuarios reales para validar navegación intuitiva y experiencia de usuario satisfactoria.

10. Conclusiones

El desarrollo de este proyecto ha sido una experiencia de aprendizaje integral que ha permitido aplicar y consolidar todos los conocimientos adquiridos durante el primer trimestre del ciclo de Desarrollo de Aplicaciones Multiplataforma.

Se ha conseguido crear un portfolio profesional completamente funcional utilizando HTML5, CSS3 y JavaScript vanilla, demostrando que es posible desarrollar interfaces modernas, atractivas y con excelente experiencia de usuario sin necesidad de frameworks o librerías externas. Esto refuerza la importancia de dominar los fundamentos antes de recurrir a herramientas más avanzadas.

Entre los logros más destacados se encuentran: la implementación de un sistema de diseño coherente mediante variables CSS, la creación de animaciones complejas que mejoran la experiencia sin comprometer el rendimiento, el desarrollo de un diseño completamente responsive que funciona perfectamente en todos los dispositivos, y la aplicación de principios de accesibilidad y semántica HTML.

Este proyecto no solo cumple con los objetivos académicos establecidos, sino que también representa una herramienta práctica y profesional que utilizaré en mi búsqueda de prácticas y futuras oportunidades laborales. Demuestra competencias técnicas sólidas, capacidad de aprendizaje autónomo, y habilidad para crear productos digitales de calidad.

11. Líneas de Trabajo Futuro

Aunque el proyecto cumple satisfactoriamente con todos los requisitos, se identifican las siguientes mejoras potenciales para futuras iteraciones:

  • Modo Oscuro/Claro: Implementación de un sistema de temas que permita al usuario alternar entre modo claro y oscuro, mejorando la accesibilidad y preferencias personales.
  • Internacionalización: Añadir soporte multiidioma (español/inglés) para ampliar el alcance del portfolio a oportunidades internacionales.
  • Backend y CMS(solo si Luismi nos deja): Integración con un sistema de gestión de contenidos para actualizar proyectos dinámicamente sin modificar código HTML.
  • PWA (Progressive Web App): Convertir el portfolio en una PWA para permitir instalación en dispositivos y funcionamiento offline.
  • Optimizaciones Avanzadas: Implementar lazy loading para imágenes, code splitting, y minificación avanzada de recursos para mejorar aún más el rendimiento.
  • Analytics: Integración con Google Analytics para entender mejor el comportamiento de los visitantes y optimizar la experiencia de usuario basándose en datos reales.
